BpForms notation

Overview

BpForms is a notation for representing biopolymer forms as FASTA sequences augmented with modified residues described by one or more attributes in square brackets. Multiple attributes are separated by "|".

Examples

  • [id: "dI" | name: "deoxyinosine"]ACGC: represents deoxyinosine at the first position
  • AC[id: "dI" | name: "deoxyinosine"]GC: represents deoxyinosine at the third position
  • ACGC[id: "dI" | name: "deoxyinosine"]: represents deoxyinosine at the last position

Describing the structures of modified bases

The structure attribute can describe the chemical structure of modified residues. This should be an InChI-encoded string. Each modified residue can have one structure.

Examples

[id: "dI" | structure: InChI=1S
    /C10H12N4O4
    /c15-2-6-5(16)1-7(18-6)14-4-13-8-9(14)11-3-12-10(8)17
    /h3-7,15-16H,1-2H2,(H,11,12,17)
    /t5-,6+,7+
    /m0
    /s1
]

Describing uncertainty about the identity and location of bases

BpForms can represent two types of uncertainty in the structures of biopolymer forms.
  • The delta-mass delta-charge attributes can describe uncertainty in the chemical identities of modified residues.
  • The position attribute can describe uncertainty in the positions of modified residues.

Examples

  • [id: "dAMP" | delta-mass: 1 | delta-charge: 1]: indicates that the presence of an additional hydrogen atom whose exact location is not known.
  • [id: "dI" | position: 2-3]: indicates that deoxyinosine may occur anywhere between the second and third position.

Describing metadata about modified bases

BpForms can represent several types of metadata about modified residues.
  • The id and name attributes can be used to provide human-readable labels for modified residues. Only one id and one name is allowed per residue.
  • The synonym attribute can be used to provide additional human-readable labels. Each residue can have multiple synonyms.
  • The identifier attribute can be used to provide references to entries in external databases. Each residue can have multiple identifiers. The namespaces and ids of identifers must be separated by "/".
  • The comments attribute can be used to describe additional information about each residue. Each residue can only have one comment.

Examples

  • [id: "dI" | name: "deoxyinosine"]
  • [id: "dI" | synonym: "deoxyinosine" | synonym: "2'-deoxyinosine"]
  • [id: "dI" | identifier: chebi/CHEBI:28997 | identifier: pubchem.compound/65058]
  • [id: "dI" | comments: "A purine 2'-deoxyribonucleoside that is inosine ..."]
